<TYPESCRIPT : RAZÃO>

O motivo do Typescript é conseguir de maneira mais eficiente
identificar e tratar erros de tipagem, ou seja, ele ajuda na
versão final do código, corrigindo possiveis problemas de tipos e
dando ferramentas para garantir que o código final não esteja
falhando em algum momento da execução.

Typescript e Javascript se escrevem da mesma forma, porem o 
Typescript tem novos metodos para auxilio no tratamento da 
tipagem dos valores processados.

<COMANDOS DE TIPAGEM>
let numero = window.document.getElementById('numb') as HTMLInputElement;

as HTMLInputElement;

Utilizamos esse trecho acima para definir que a variavel
numero está recebendo exatamente um "tipo de entrada HTML",

dessa forma, na hora de executar o código, não haverá duvidas
que está correto o processamento para entrada da variavel numero.

Agora considere a seguinte função ::

function somar(n1, n2){
    return n1 + n2
}

No typescript, isso vai gerar um alerta, pois n1 e n2 não foram
especificados como numerico, e isso pode ocarionar um problema...

Para solucionar isso, adicionamos o seguinte ::

function somar(n1: number, n2: number){
    return n1 + n2
}

seguindo essa ideia, caso não fosse uma soma e sim um string,
poderiamos utilizar ::

n1: string

Agora considere o seguinte caso ::

res.innerHTML = somar(10, 20) 

:: vai aletar possivel erro, porque
um elemento HTML sempre espera um retorno string, sendo que o 
retorno nesse caso seria numerico, então precisamos converter o
resultado final em string conforme o exemplo abaixo ::

res.innerHTML = somar(10, 20).toString()
