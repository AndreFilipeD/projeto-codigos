Vamos considerar a função e o objeto abaixo ::

function resumo(usuario:{nome: string, idade: number}){
    return `Olá ${usuario.nome}, você tem ${usuario.idade} anos.`
}

let u = {
    nome: 'andré',
    idade: 90
}

Vamos receber um objeto para ser processado dentro da função resumo,
mas perceba que para definir o tipo de cada parâmetro
do objeto, precisamos meio que criar um objeto de parâmetros
dentro da declaração da função (Destacado em amarelo )

Dessa forma, ele já verifica cada item do objeto recebido,
para ter certeza que os tipos estão corretos antes de compilar;
E se idade fosse um parâmetro opcional? exemplo ::

function resumo(usuario:{nome: string, idade?: number}){

    if(usuario.idade !== undefined){ //CASO TENHA
        return `Olá ${usuario.nome}, você tem ${usuario.idade} anos.`
    }else{ // CASO NÃO TENHA
        return `Olá ${usuario.nome}, tudo bem?`
    }
    
}

:: Perceba o interrogação destacado em amarelo, ele diz que,
pode ocorrer de a propriedade idade ser undefined,
que não será considerado um problema, já que o codigo está
preparado para isso.
