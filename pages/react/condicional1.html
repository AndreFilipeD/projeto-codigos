Podemos utilizar condições na hora de renderizar o conteudo
em react conforme os seguintes exemplos abaixo ::

type Props = {
    phase: string;
    author?: string; //author está como opcional
}

export const Card = ({ phrase, author}: Props) => {
    if(author) {
        return(
        <>
            <h3>Frase: "{phrase}" </h3>
            <p>Autor: "{author}" </p>
        </>   
    );
    }else{
        return(
        <>
            <h3>Frase: "{phrase}" </h3>
            <p>Autor: "DESCONHECIDO" </p>
        </>   
    );
    }
}

:: Veja como ele verifica se, ao ser criado o componente, foi
informado algum author, if true, ele coloca o autor no local,
if false, ele coloca DESCONHECIDO.

Claro que foi um exemplo basico, mas podemos melhorar assim ::

return(
        <>
            <h3>Frase: "{phrase}" </h3>
            <p>Autor: "{author ? author : "DESCONHECIDO"}" </p>
        </>   
);

:: Utilizando condicional ternário, na mesma linha conseguimos
estruturar essa substituição no caso do componente não receber
o valor; mas dá pra ficar ainda melhor por incrivel que pareça...

<p>Autor: "{author ?? "DESCONHECIDO"}" </p>

:: Com os 2 interrogações, ele faz exatamente a mesma coisa..
