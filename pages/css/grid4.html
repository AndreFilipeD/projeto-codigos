<EXPERIMENTOS COM GRID TEMPLATE AREAS>

• Podemos definir um nome para cada elemento dentro
do container, e assim organizando-os previamente.

.red{ grid-area: fogo;}
.green{ grid-area: terra;}
.blue{ grid-area: agua;}
.yellow{ grid-area: raio;}

.container{ grid-template-areas: "raio agua" "terra fogo";}


    
 

    
 

    
 

    
 


Veja como a sequência foi alterada para o padrão criado.

Agora vamos alongar o .container utilizando
grid-template-columns: repeat(3, 50px); que é equivalente a 
50px 50px 50px, e depois adicionar 2 elementos que não
declaramos propositalmente, gelo e vento

    
 

    
 

    
 

    
 


.container{
    grid-template-areas: "raio vento terra"
    "fogo gelo agua";
}
• Como o gelo e o vento não foram definidos em nenhum bloco,
é deixado o espaço vazio.

• Também pode utilizar repeat(auto-fill, 50px) para que seja
definido dinamicamente quantas colunas ou linhas terá baseado
na possibilidade de incluir mais 50px no tamanho da caixa/tela
onde está apresentado.

Veja o que acontece ao definir um mesmo nome
para dois espaços:


    
 

    
 

    
 

    
 


.container{
    grid-template-areas: "raio raio terra"
    "fogo agua agua";
}
:: Dessa forma, os elementos são esticados, mas para isso,
os mesmos precisam estar com align-items e justify-items
como "stretch"(default), e sem nenhum tamanho fixo definido.
